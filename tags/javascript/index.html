이벤트

이벤트 드리븐 프로그래밍이란?

이벤트가 발생할 때 호출되는 함수인 이벤트 핸들러를 이용하여 이벤트 중심으로 프로그램의 흐름을 제어하는 방식을 말한다.

이벤트 타입

  • 마우스 이벤트

    • click : 클릭할 때
    • dbclick : 더블 클릭할 때
    • mousedown : 마우스 버튼을 누를 때
    • mouseup : 누르던 마우스 버튼을 놓았을 때
    • mousemove : 마우스를 움직일 때
    • mousenter : 마우스를 요소안에 이동했을 때
    • mouseleave : 마우스가 요소 안에서 밖으로 이동했을 때

  • 키보드 이벤트

    • keydown : 문자, 숫자, 특수 문자 키를 누를 때
    • keyup : 눌렀던 키를 놓았을 때
    • keypress : 모든 키를 눌렀을 때 한번만!

  • 포커스 이벤트

    • focus : tab으로 요소가 포커스를 받았을 때
    • blur : 포커스 받은 요소가 포커스를 잃었을 때

  • 폼 이벤트

    • submit : submit 버튼을 눌렸을 때
    • reset : reset 버튼을 눌렀을 때

  • 값 변경 이벤트

    • input : input, select, textarea 요소에 값이 입력되었을 때
    • change : input 요소에 입력하지 않고 포커스가 나갔을 때
    • readystatechange : HTML 문서의 로드와 파싱 상태를 나타내는 readyState 프로퍼티 값이 변경될 때

  • DOM 뮤테이션 이벤트

    • DOMContentLoaded : HTML 문서의 로드와 파싱이 완료되어 DOM 및 CSSOM 생성이 완료 되었을 때

  • 뷰 이벤트

    • resize : 브라우저 윈도우의 크기가 변할 때
    • scroll : 웹 페이지를 스크롤 할 때

  • 리소스 이벤트

    • load : DOM 이벤트 이후에 모든 리소스를 로딩 받았을 때
    • unload : 리소스가 언로드 될 때
    • abort : 리소스 로딩이 중단되었을 때
    • error : 리소스 로딩을 실패하였을 때

이벤트 핸들러 등록

이벤트 핸들러란? 이벤트가 발생하였을 때 브라우저에 의해 실행될 함수를 이벤트 핸들러라고 한다.
이벤트 핸들러를 이벤트에 매칭하여 등록하면 해당 이벤트 발생시 해당 함수를 실행한다.

  • 어트리뷰트 방식
    HTML 요소의 이벤트에 해당하는 어트리뷰트에 이벤트 핸들러를 등록하는 방법.
    접두사 on과 이벤트 타입으로 이루어져있다.

  • 프로퍼티 방식
    HTML 요소 이벤트에 해당하는 프로퍼티 키에 대해 이벤트 핸들러를 값으로 할당하는 방법.
    접두사 on과 이벤트 타입으로 이루어져있다.

  • addEventListener 메소드 방식
    EventTarget.prototype.addEventListener 메소드를 사용해서 첫번째 인수로 이벤트 타입을 두번쨰 인수로 이벤트 이벤트 핸들러를 전달하면 등록이 된다. 세번째는 옵션으로 caption 사용 여부를 묻는다.
    동일한 이벤트 핸들러를 중복등록하면 하나만 등록된다.

이벤트 핸들러 제거

EventTarget.prototype.removeEventListener 메소드를 사용하면 addEventListener 메소드로 등록한 이벤트 핸들러를 삭제할 수 있다. 주의해야할 점은 addEventListener 메소드에 준 인수를 그대로 주어야한다.

이벤트 객체

이벤트가 발생하면 이벤트 정보를 담고 있는 객체가 만들어지는데 이를 이벤트 객체라고 한다. 이벤트 객체는 이벤트 타입에 따라 프로퍼티가 다른 객체를 만든다.

  • 이벤트 객체의 공통 프로퍼티

    • type : 이벤트 타입
    • target : 이벤트를 발생 시킨 DOM 요소
    • currentTarget : 이벤트 핸들러가 바인딩 된 DOM 요소
    • eventPhase : 이벤트 전파 단계 (0:이벤트 없음, 1: 캡처링 단계, 2: 타깃 단계, 3: 버블링 단계)
    • bubbles : 이벤트를 버블링으로 전파하는지 여부
    • cancelable : preventDefult 메소드를 호출하여 이벤트 기본 동작 취소 가능 여부
    • defultPrevented : preventDefult 메소드를 호출하여 이벤트 취소 가능 여부
    • isTrusted : 사용자가 발생시킨 이벤트인지 확인
    • timeStemp : 이벤트가 발생한 시간

  • 마우스 정보 취득

    • 마우스 포인터 좌표 정보 프로퍼티 : screenX/screenY, clientX/clientY, pageX,pageY, offsetX/offsetY
    • 버튼 정보 프로퍼티 : altKey, ctrlKey, shiftKey, button

  • 키보드 정보 취득

    • 키보드 정보 프로퍼티 : altKey, ctrlKey, shiftKey, metaKey, Key, keyCode

이벤트 전파

트리 상에서 존재하는 DOM 요소들에 이벤트가 발생하면 트리 구조에 따라 이벤트가 전파된다. 이를 이벤트 전파라고 한다.
전파과정에 따라

  • 캡처링 단계 : 이벤트가 상위 요소에서 하위 요소 방향으로 전파
  • 타깃 단계 : 이벤트가 이벤트 타겟에 도달
  • 버블링 단계 : 이벤트가 하위 요소에서 상위 요소 방향으로 전파

이벤트 위임

모든 요소에 동일한 이벤트 핸들러를 가지면 퍼포먼스가 나쁘기 때문에 하나의 상위 요소에 이벤트 핸들러를 등록하는 방법을 이벤트 위임이라고 한다.

기본 동작 변경

HTML 요소들은 각각 저마다의 기본 동작을 갖는다. 이 기본 동작을 조작할 수 있다.

  • 기본 동작 중지 : 기본동작을 해제하기 위해 preventDefult 메소드를 사용하면 된다.

  • 이벤트 전파 방지 : stopPropagation 메소드를 사용한다.

이벤트 핸들러 내부에서의 this

  • 어트리뷰트 방식 : 전역 객체 window

  • 프로퍼티와 addEventListener 메소드 방식 : 이벤트를 바인딩한 DOM 요소

이벤트 핸들러에 인수 전달

이벤트 핸들어 안에서 즉시실행함수로 만들어 인수를 전달할 수 있다.

커스텀 이벤트

이벤트 객체는 Event, UIEvent, MouseEvent와 같은 생성자 함수로 만들 수 있다.

DOM

DOM이란?

HTML 문서의 계층적 구조와 정보를 표현한 트리 자료 구조. 브라우저 사이드 API 를 제공하여 프로퍼티와 메소드를 사용하여 제어할 수 있다.

노드란?

노드는 HTML 요소들이 랜더링 엔진에 의해 파싱되어 DOM을 구성하는 요소가 되는데 이를 노드라고 한다.

노드의 객체 타입

  • 문서 노드
    DOM 트리 최상위에 존재하는 루트 노드. document 객체를 말한다.
    전역 객체 window의 document 프로퍼티에 바인딩 되어 있다.

  • 요소 노드
    HTML 요소를 가르키는 객체이다. HTML 요소 간의 중첩 관계를 가지고 부자 관계를 가지며 정보를 구조화한다.

  • 어트리뷰트 노드
    HTML 요소의 어트리뷰트를 가르키는 객체이다. 지정된 HTML 요소 노드와 형제 관계를 갖는다.

  • 텍스트 노드
    HTML 요소의 텍스트를 가르키는 객체이다. 요소 노드의 자식 노드이며 DOM 트리의 최종단인 리프 노드이다.

노드의 상속 구조

DOM을 구성하는 노드 객체는 프로토타입에 의한 상속 관계를 갖는다.
Object - EventTarget - Node 가 기본으로 갖는 프로토타입 체인이며 그 자식 관계로 노드에 따라
문서 노드는 Document - HTMLDocument
요소 노드는 Element - HTMLElement - HTML(태그 이름)Element
어트리뷰트 노드는 Attr
텍스트 노드는 CharacterData - Text/Conment
라는 노드 객체의 상속 구조를 갖는다.
이 상속관계에 따라 해당 노드 객체는 브라우저에서 필요한 기능을 제공하는 DOM API 를 사용할 수 있다.

요소 노드 취득

HTML의 구조나 내용, 스타일을 조작하기 위해서는 해당 요소의 노드를 취득해야한다.

  • id 요소로 취득
    Document.prototype.getElementById 메소드를 사용하여 인수로 전달한 id 값을 갖는 하나의 요소를 탐색하여 반환한다.

    • 당연히 id는 중복되지 않는 값이기 때문에 1개만 있어야하지만 만약 동일한 id 값을 갖는 객체가 있다고 해도 에러 없이 해당 요소들 중 가장 첫번째 요소 객체를 반환한다.
    • 해당 id의 요소가 없을 경우, null값을 반환한다.

  • 태그 이름으로 취득
    Document.prototype.getElemenstByTagName/Element.prototype.getElemenstByTagName 메소드를 사용하여 인수로 전달한 태그 이름을 갖는 모든 요소를 탐색하여 반환된다.

    • 탐색된 값은 유사배열 객체인 HTMLCollection 객체로 반환된다.

  • class 요소로 취득
    Document.prototype.getElemenstByClassName/Element.prototype.getElemenstByClassName 메소드를 사용하여 인수로 전달한 클래스 이름을 갖는 모든 요소를 탐색하여 반환한다.

    • 탐색된 값은 유사배열 객체인 HTMLCollection 객체로 반환된다.

  • css 선택자로 취득
    Document.prototype.querySelecter/Element.prototype.querySelecter 메소드를 사용하여 인수에 전달한 css 선택자를 만족시키는 하나의 요소를 탐색하여 반환한다. 없으면 null 값을 반환
    Document.prototype.querySelecterAll/Element.prototype.querySelecterAll 메소드를 사용하면 만족하는 모든 요소를 탐색하여 반환한다.

    css 선택자에는

      • : 전체 선택자
    • (태그 이름) : 태그 선택자
    • #(id 이름) : id 선택자
    • .(클래스 이름) : 클래스 선택자
    • (태그 이름)[속성 이름 = 값] : 어트리뷰트 선택자
    • (부모 태그 이름) (자식 태그 이름) : 후손 선택자 : 부모 밑으로 상속되는 모든 자식
    • (부모 태그 이름) > (자식 태그 이름) : 자식 선택자 : 부모 바로 밑에 있는 자식
    • (형제 태그 이름) + (찾을 형제 태그 이름) : 인접 형제 선택자 : 검색 기준인 형제 요소에서 바로 뒤에 위치하는 인접한 형제
    • (형제 태그 이름) ~ (찾을 형제 태그 이름) : 일반 형제 선택자 : 검색 기준인 형제 요소 뒤에 오는 형제 태그들 모두
    • (태그 이름):(가상 클래스 이름) : 가상 클래스 선택자 : 해당 상태인 태그

HTMLCollection 객체와 NodeList 객체

DOM API가 검색한 여러개의 노드를 결과값으로 반환하기 위해 만드는 유사배열 객체이다.

  • HTMLCollection 객체
    태그 이름으로 취득하는 경우나 클래스로 취득하는 경우에 반환되는 값으로서 노드 객체의 변화를 실시간으로 반영한다.(살아있는 객체)
    살아있는 객체 문제를 해결하기 위해서 배열로 변환하여 사용하는 것을 추천한다.

  • NodeList 객체
    css 선택자로 취득하는 경우에 반환되는 값이나 자식 요소를 찾는 childNodes 프로퍼티가 반환한 값이다.
    css 선택자로 취득하는 경우에는 노드 객체의 변화를 반영하지 않지만, childNode 프로퍼티로 반환되면 노드 객체의 변화를 반영하는 살아있는 객체가 된다.

노드 탐색

어떤 요소를 취득하면 그 요소를 통해 부모, 형제 자식 요소에 접근할 수 있다. 이렇게
해당 노드는 노드 탐색을 위한 접근자 프로퍼티를 갖는다. 읽기 전용 값이다. 즉, 값을 할당하면 에러 없이 무시된다.

  • parentNode : 부모 요소
  • previousSibling/previousElementSibling : 바로 앞에 있는 형제 요소 (텍스트 노드 포함/미포함)
  • nextSobling/nextElementSibling : 바로 뒤에 있는 형제 요소 (텍스트 노드 포함/미포함)
  • firstChild/firstElementChild : 첫번째 자식 요소 (텍스트 노드 포함/미포함)
  • lastChild/lastElementChild : 마지막 자식 요소 (텍스트 노드 포함/미포함)
  • childNodes/children : 자식 요소들 (텍스트 노드 포함/미포함)

공백 텍스트 노드

HTML 문서에서 태그 사이에 개행문자가 포함되있으면 그 공백을 의미하는 텍스트 노드를 생성한다. 따라서 검색시에 유의해야한다.

자식 노드 확인

Node.prototype.hasChildNodes 메소드로 확인할 수 있다. 하지만 이 메소드는 텍스트 노드를 포함하여 자식 노드의 존재를 확인한다.
따라서 요소 노드만을 확인하고 싶은 경우에는 Node.prototype.children.length 또는 Node.prototype.childElementCount 프로퍼티로 확인하면 된다.

텍스트 노드 확인

해당 노드에 자식으로 텍스트 노드가 연결되어 있기 때문에 firstChild나 childNodes로 접근할 수 있다.

탐색 가능 여부 확인

Element.prototype.matches 메소드로 검색할 특정 노드를 선택자의 형태로 전달하면 확인 가능하다.

노드 정보 취득

  • Node.prototype.nodeType
    노드 객체의 종류를 나타내는 상수를 반환한다.

    • 요소 노드 : 1
    • 텍스트 노드 : 3
    • 문서 노드 : 9

  • Node.prototype.nodeName
    노드 이름을 문자열로 반환한다.

    • 요소 노드 : 대문자 문자열로 태그 이름 반환
    • 텍스트 노드 : ‘#text’ 문자열 값을 반환
    • 문서 노드 : ‘#document’ 문자열 값을 반환

요소 노드 텍스트 조작

  • nodeValue
    텍스트 노드의 값을 반환한다. 이때 호출하는 대상이 텍스트 노드가 아닌 경우 null을 반환한다.

  • textCotent
    해당 요소의 모든 컨텐츠 영역 내에 텍스트를 전부 반환한다. 하위 요소에 텍스트 노드의 값을 전부 반환한다.

Symbol 타입

Symbol 타입 데이터란?

Symbol 타입은 중복되지 않는 유일한 값을 생성할 때 사용한다.

2가지 기능으로 사용 가능하다.

  • 객체의 프로퍼티의 키가 의미를 갖고 있어 프로퍼티 값이 그 키의 값이라는 것만을 명시하고 싶을 때 만드는 값으로 사용
  • 객체의 프로퍼티 키로 나중에 새로 추가하는 프로퍼티 키와 중복될 위험성이 있을 때 충돌하는 것을 방지하기 위해서 사용

따라서 Symbol 타입 데이터를 만드는 순간 그 값은 어떤 값하고도 중복되지 않는다.

똑같은 인수를 넣어도 만드는 순간 유일한 값이 된다.

Symbol 타입 데이터는 다음과 같은 특징을 갖는다.

  • 암묵적으로 문자열이나 숫자 타입으로 변환되지 않는다. 단 불러언 타입으로는 변환된다.

Symbol 타입 데이터 생성

  • Symbol 함수
    Symbol 함수를 호출하여 만들 수 있다.
    생성자 함수처럼 보이지만 아니다. new 연산자와 함께 호출하면 TypeError를 출력한다. constructor가 없다고 나온다.

  • Symbol.for 메소드
    Symbol.for 메소드는 인수로 들어온 문자열을 갖고 만든 심볼 값이 있는지 검색한다.
    있으면 해당 심볼 값을 반환하고 없으면 심볼 값을 만들어 반환한다.
    단, 주의할 점은 Symbol.for 메소드는 Symbol 함수로 만든 심볼 값을 검색하지 않는다.

Symbol 함수와 Symbol.for 메소드의 차이

Symbol 함수로 생성한 심볼 값은 검색할 수 있는 심볼 키를 지정하지 않기에 심볼 값 저장소인 전역 심볼 레지스트리에 저장하지 않는다.
하지만 Symbol.for 메소드로 생성하면 심볼 키를 갖는 심볼 값을 생성하기에 전역 심볼 레지스트리에 저장되며, 그 값을 공유할 수 있다.

Symbol과 프로퍼티

심볼 값을 프로퍼티 키로 사용하기 위해서는 대괄호 표기법으로 정의해야하며, 참조시에도 대괄호 표기법으로 참조해야한다.
주의할 점은, Symbol 함수로 정의한 키는 어떤 방식으로도 참조할 수 없다는 점이다. 참조하기 위해서는 Symbol.for로 정의하고 검색해야한다.
따라서, Symbol함수로 정의한 키는 프로퍼티를 숨길 수 있다. 하지만 Object.getOwnPropertySymbols 정적 메소드를 사용해서 해당 객체의 심볼 값를 참조할 수 있다.
Object.getOwnPropertySymbols 정적 메소드는 인수로 받는 객체에 모든 심볼 프로퍼티 키를 요소로 갖는 유사배열객체로 반환한다.

Symbol과 표준 빌트인 객체 확장

표준 빌트인 객체에 프로퍼티를 추가하는 것은 후에 업데이트가 되면서 프로퍼티 키가 중복이 될 가능성이 있기 때문에 권장하지 않지만, 심볼 값을 키로 갖는 프로퍼티로 만들 경우 중복될 일이 없기에 사용할 수 있다.

배열이란?

여러 개의 값을 순차적으로 나열한 자료 구조이다. 주로 관계가 있는 여러 개의 값을 묶어두는 데 사용한다.

  • 요소
    배열이 갖고 있는 값.
  • 인덱스
    배열의 요소가 자신의 위치를 나타내는 0 이상의 정수. 인덱스로 값을 찾는다.
  • length
    배열의 길이를 나타낸다.
    length는 반드시 배열 요소의 갯수를 나타내는 것이 아니다.

자바스크립트와 다르게 다른 언어는 요소들의 타입을 동일하게 맞추어주어야 하지만 자바스크립트는 요소들의 타입이 달라도 된다.
동일한 타입의 요소를 갖고 시작 메모리 주소를 통해 빈틈없이 연속적으로 메모리에 요소들이 저장되있는 배열을 밀집 배열이라 하며,
자바스크립트처럼 연속적으로 메모리에 저장되있지 않은 배열을 희소 배열이라 한다.

밀집 배열과 희소 배열

밀집 배열은 연산을 통한 정렬된 특정 요소의 접근을 효율적으로 할수 있지만, 배열에 요소를 삽입이나 삭제를 해야하는 경우 삽입이나 삭제한 메모리 이후의 요소들을 전부 이동시켜야하는 단점이 있다. 희소 배열은 밀집 배열의 장단점이 반대이다.

자바스크립트와 배열

사실 자바스크립트는 인덱스를 프로퍼티 키로 갖으며, length 프로퍼티를 갖고, Array.prototype 메소드를 사용할 수 있는 객체이다.
따라서, length에 값을 할당할 수 있고, 그 값은 배열의 요소의 수보다 작거나 클수 있다.

  • length 값이 배열 요소들의 수보다 작으면 배열의 길이가 작아진다. 그 이상의 요소들을 순회할 수 없다는 뜻이다.
  • length 값이 배열 요소들의 수보다 클 때는 배열의 길이가 늘어나지는 않는다. 실제 배열에는 아무런 변화가 없고 단지 length 값이 바뀔 뿐이다.

또한 자바스크립트는 비어있는 요소를 넣을 수 있다. 이를 희소 배열이라고 한다.

유사 배열 객체

마치 배열처럼 인덱스로 프로퍼티 값에 접근할 수 있고, length 프로퍼티를 갖는 객체.
length 프로퍼티가 있음으로 for 문으로 순회할 수 있다.
단, 배열 객체와 차이는 자바스크립트에서 제공하는 Array.prototype 메소드를 사용할 수 없다는 것이다.

    전달된 유사 배열 객체 또는 이터러블 객체를 분해하여 각각의 값을 요소로 갖는 배열을 생성한다.
    유사 배열 객체란? 객체지만 배열처럼 인덱스를 키로 갖고 length 프로퍼티를 갖는 객체를 말한다.
    이터러블 객체란? 유사 배열 객체처럼 length 프로퍼티를 갖고 순회할 수 있는 객체를 말한다.

배열 요소 참조/추가/갱신/삭제

  • 참조시에 대괄호 표기법을 사용한다. 대괄호 안에는 인덱스가 와야한다.
  • 추가 및 갱신은 해당 배열의 길이와 같은 숫자의 인덱스 번호를 대괄호 표기법으로 참조하고 값을 할당하면 추가할 수 있다. 배열의 길이보다 큰 수의 인덱스 번호로 추가하면 희소 배열이 된다.
  • 삭제는 프로퍼티 삭제처럼 delete 연산자를 통해 삭제할 수 있지만 해당 인덱스가 통채로 삭제되며 length 값도 변하지 않는다.
    따라서, Array.prototype.splice 메소드를 사용해서 삭제해야한다.

ES6 함수의 추가 기능

함수의 구분

ES6에서 사용목적에 따라 함수 구분한다.

  • 일반 함수
  • 메소드
  • 화살표 함수

일반 함수

ES6이전에 사용하던 함수 선언문이나 함수 표현식으로 정의된 함수이다.
ES6에서 오직 이 일반 함수만이 constructor 함수임으로 일반 함수만 new 연산자로 호출할 수 있다.

메소드

ES6에서는 메소드 축약 표현으로 정의된 함수만을 메소드로 취급한다.
그리고 이렇게 정의된 메소드만 메소드가 바인딩된 객체를 가르키는 내부슬롯 [[HomeObject]]을 갖는다. 따라서 super 키워드는 이 메소드만 사용 가능하다.

화살표 함수

function 키워드 대신에 화살표(=>)를 사용하여 정의한 함수이다.
(매개 변수들…) => {함수 몸체} 로 이루어진다.
화살표 함수는 다음과 같은 특징을 갖는다.

  • 함수 이름을 선언할 수 없다.

  • 매개 변수
    매개 변수가 한개인 경우에만 ()를 생략 가능하다.
    오직 매개 변수가 한개인 경우만 생략 가능하다. 없으면 써주어야한다.
    또한, 일반함수와 다르게 중복된 매개변수를 사용할 수 없다.

  • 함수 몸체
    함수 몸체가 한 줄의 문으로 구성되면 {}를 생략하고 바로 사용 가능하며 해당 문을 반환한다.
    한줄이 아닌 경우에는 생략할 수 없고, 반환값이 있다면 명시적으로 반환해야한다.
    객체 리터럴만을 반환하는 경우 {}를 생략하고 ({반환할 객체 리터럴})의 형식으로 사용가능하다.

  • this, arguments, super, new.target 바인딩을 갖지 않는다.

    • this
      화살표 함수 내에서 this를 참조하면 상위 컨텍스트의 this를 참조하게 된다.
      this 바인딩을 갖지 않기 때문에, Function.prototype.apply/call/bind로 this 바인딩해줄 수 없다.

    • super/ arguments
      this와 마찬가지로 바인딩 되어있지 않기에 참조시 상위 컨텍스트의 super/arguments를 참조한다.

Rest 파라미터

매개 변수 이름 앞에 …를 붙여서 정의한 매개 변수이다. 전달받은 인수들을 배열로 전달 받는다.
Rest 파라미터가 있음으로 arguments 객체를 배열로 변환하는 번거로움을 피할 수 있다.

매개변수의 기본값

일반적으로 매개 변수를 정의하고 따로 초기화을 하지 않으면 undefined 값으로 초기화된다.
하지만 ES6에서 매개변수에 바로 할당문을 사용함으로서 초기화를 간소화 할 수 있다.

클래스

ES6부터 도입된 클래스 기반 프로그래밍 언어와 유사한 새로운 객체 생성 메커니즘이니다.
class 키워드를 사용하여 정의하며, 생성자 함수로 마찬가지로 파스칼 케이스를 사용하는 것이 일반적이다.

함수와 마찬가지로 클래스도 일급 객체임으로 표현식으로 정의할 수도 있다.

클래스의 특징

  • 클래스 호이스팅
    클래스는 정의 이전에 참조할 수 없다.
    참조시, ReferenceError를 출력한다. let, const 처럼 작동한다.

  • 생성자 함수로 평가
    클래스는 함수로 평가된다.
    인스턴스를 생성하기 위해서 반드시 new 연산자와 함께 사용해야한다.
    new 연산자를 사용하지 않고 호출 했을 시, TypeError를 출력한다.

클래스에서 정의하는 메소드

일반적으로 클래스 몸체에 0개 이상의 메소드만을 선언할 수 있다.
정의할 수 있는 메소드는 3가지가 있다.

  • constructor(생성자)
    인스턴스를 생성하고 초기화하기 위한 메소드이다. 클래스 안에서 없거나 단 한개만 정의해야 한다.
    2개 이상 정의할 시 syntaxError를 출력한다.
    생략하면 암묵적으로 디폴트 constructor가 정의된다.

    해당 메소드 안에 this는 생성할 객체를 가르킨다.

    constructor 메소드 몸체 안에 return문을 쓰면 반환할 값이 객체일 때는 해당 객체를 반환하며 원시값이면 무시한다.
    하지만 this가 아닌 다른 값을 반환하는 것은 클래스의 기본동작을 훼손하기 때문에 return 문은 반드시 생략해야한다.

  • 프로토타입 메소드
    클래스 몸체에 정의한 메소드(메소드 축약 표현)는 생성할 인스턴스 객체의 프로토타입 객체의 메소드로 추가된다.

  • 정적 메소드
    클래스 몸체에 strtic 키워드를 사용하고 메소드를 정의하면 정적 메소드로 해당 class의 메소드가 된다.

클래스에서 정의하는 메소드의 특징

  • function 키워드를 생략한 메소드 축약 표현을 사용한다.
  • 정의할 때 콤마(;)가 필요없다.
  • 암묵적으로 strict 모드로 실행된다.
  • 열거할 수 없다. [[Enumerable]]값이 false이다.
  • [[Construct]]를 갖지 않는 non-costructor 함수이다.

클래스에 의한 인스턴스 생성과정

  1. 인스턴스 생성과 this 바인딩
    new 연산자와 함께 클래스를 호출하면 constructor 메소드의 내부 코드가 실행되고 우선 암묵적으로 빈 객체를 만들며 이 객체를 this에 바인딩한다.

  2. 인스턴스 초기화
    this에 바인딩 되있는 객체에 프로퍼티를 추가한다. 클래스를 호출시 입력했던 인수를 constructor 매개변수에 전달하여 받은 값으로 인스턴스의 프로퍼티 값을 초기화한다.

  3. 프로토타입 / 정적 메소드 추가
    프로토타입 메소드는 생성될 인스턴스의 프로토타입 객체에 추가하며, 정적 메소드는 해당 클래스에 추가된다.

  4. 인스턴스 반환
    this에 바인딩된 객체를 반환한다.

프로퍼티 정의

인스턴스의 프로퍼티는 constructor 메소드 내부에서 생성자함수 처럼 프로퍼티 동적 생성을 통해 생성한다.
접근자 프로퍼티도 정의할 수 있다.

클래스 필드 정의(제안)

클래스 필드란 클래스 기반 객체 지향 언어에서 클래스가 생성할 인스턴스의 프로퍼티를 가르키는 용어이다.
자바스크립트는 원래 constructor 안에서 프로퍼티 동적 생성을 통해서만 프로퍼티를 생성할 수 있다. 클래스 몸체에서 정의하려고 하면 SyntaxError를 출력한다.

하지만, 현재(2020.03)에 클래스 필드 정의를 의미하는 “Class field declarations”가 stage 3에 제안 되어있다.
최신 버전의 브라우저와 node.js에서는 클래스 필드 정의를 할 시 오류를 출력하지 않는다.

클래스 필드 정의를 사용하여 프로퍼티를 정의할 때 this는 constructor 메소드 안에서만 유효함으로 this를 사용하면 안된다. 하지만 해당 프로퍼티를 참조하는 경우에는 반드시 this를 사용해야한다.

클래스 필드에 초기값을 할당하지 않으면 undefined를 갖는다.
하지만 후에 초기화를 해주어야한다면 constructor 메소드에서 할당을 해주어야하기 때문에 클래스 필드 정의를 할 필요가 없다.

클래스 필드에 함수를 할당하면 프로토타입 메소드가 아닌 인스턴스의 메소드가 된다.

private 필드 정의(제안)

자바스크립트에는 클로저의 개념을 사용해서 다른 객체 지향 언어의 private 키워드의 기능을 흉내낼 수 있다.
차이는 private된 변수에 접근하면 에러가 아닌 undefined가 반환된다.
하지만, private 필드 정의도 클래스 필드 정의와 마찬가지로 제안 되어있다.
private 필드 정의는 해당 필드 정의 선두에 ‘#’를 붙여주면 된다.
이렇게 되면 해당 클래스 필드는 클래스 내부에서만 참조가능하다.(접근자 프로퍼티로 접근하는 것은 제외)

static 필드 정의(제안)

정적 프로퍼티도 클래스 필드 앞에 static 키워드를 사용하여 정의하면 가능하다.

상속에 의한 클래스 확장

기존의 클래스를 상속받아 새로운 클래스를 만들 수 있다. extends 키워드를 통해 클래스 상속을 통해 다른 클래스를 확장할 수 있다.
상속을 통해 확장된 클래스를 서브 클래스라고 부르고 서브 클래스에게 상속된 클래스를 수퍼 클래스라고 한다.
서브 클래스는 파생 클래스 또는 자식 클래스, 수퍼 클래스는 베이스 클래스 또는 부모 클래스라고 한다.

  • extends 키워트
    class ‘서브 클래스’ extands ‘수퍼 클래스[또는 생성자 함수]’ {} 의 형식으로 사용한다.

  • super 키워드
    super 키워드는 서브 클래스에서 수퍼 클래스를 호출하거나 참조할 때 사용된다.

    • super 호출
      호출하면 수퍼 클래스의 constructor를 호출한다.

    • super 참조
      참조하여 수퍼 클래스의 프로토타입 메소드를 호출할 수 있다.

super 키워드 사용시 주의사항

  • 호출할 시

    • 서브 클래스에서 constructor를 생략하지 않은 경우, 서부 클래스의 constructor에서 반드시 super 키워드로 호출해주어야 한다.
    • 서브 클래스에서 super 호출 전에 this를 참조할 수 없다.
    • super 키워드는 반드시 서브 클래스 constructor에서만 호출해야 한다.

  • 참조할 시

    • 서브 클래스의 프로토타입 메소드 안에서 참조할 시, 수퍼 클래스의 프로토타입를 가르킨다.
      서브 클래스는 확장될 때 내부 슬롯[[HomeObject]]에 바인딩되어있는 프로토타입 객체를 기억한다.
      주의점은, 메소드 축약 표현으로 정의된 메소드만 [[HomeObject]]를 갖는다.
    • 서브 클래스의 정적 메소드 안에서 참조할 시, 수퍼 클래스의 정적 메소드를 가르킨다.

        전역 객체에 등록해야할 변수들을 등록을 관리.

      • 선언적 환경 레코드 :
        함수 지역 스코프에 등록해야할 변수들을 등록을 관리.
        단, 전역 환경 레코드와 차이는 추가적으로 argument 객체나 매개 변수, 등을 암묵적으로 등록한다.

  • 외부 렉시컬 환경에 대한 참조 : 상위 스코프, 즉 상위 스코프를 생성하는 외부 렉시컬 환경에 대한 참조. 단방향 링크드 리스트인 스코프 체인을 구현한 것.

실행 컨텍스트의 생성과 실행 과정

  1. 전역 객체 생성 : 전역 프로퍼티와 전역 함수, 표준 빌트인 객체, 동작 환경에 따른 호스트 객체를 포함해서 생성한다.

  2. 전역 코드 평가

    1. 전역 실행 컨텍스트 생성
    2. 전역 렉팉컬 환경 생성
    3. 전역 환경 레코드 생성 : 전역 변수, 함수, 표준 빌트인 함수, 전역 빌트인 프로퍼티 및 메소드를 등록.
      1. 객체 환경 레코드 생성 :
        var 키워드로 선언한 전역 변수 및 함수를 전역 객체의 프로퍼티 및 메소드로 등록.
      2. 선언적 환경 레코드 생성 :
        let, const 키워드로 선언한 전역 변수 및 함수를 등록.
      3. this 바인딩
    4. 외부 렉시컬 환경에 대한 참조 할당

  3. 전역 코드 실행

    • 식별자 결정(식별자 검색)을 실행 컨텍스트에서 검색.
    • 함수 호출 시, 함수 코드 평가 > 함수 코드 실행 > 함수 코드 종료 순으로 동작. 함수 코드 중에 함수가 호출되면 앞의 순서를 반복.
      함수 코드 평가는 위에 전역 코드 평가에서 전역을 함수로 바꾸면 똑같이 동작한다.

  4. 전역 코드 실행 종료

실행 컨텍스트와 블록 레벨 스코프

코드 실행중 문을 실행할 때, 안에 let이나 const로 변수를 선언하면 선언적 환경 레코드를 갖는 렉시컬 환경을 새롭게 만들고 기존의 전역 렉시컬 환경을 교체한다. 그리고 기존의 전역 렉시컬 환경은 외부 렉시컬 환경에 대한 참조에 바인딩된다.
