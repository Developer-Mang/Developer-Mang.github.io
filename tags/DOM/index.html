DOM

DOM이란?

HTML 문서의 계층적 구조와 정보를 표현한 트리 자료 구조. 브라우저 사이드 API 를 제공하여 프로퍼티와 메소드를 사용하여 제어할 수 있다.

노드란?

노드는 HTML 요소들이 랜더링 엔진에 의해 파싱되어 DOM을 구성하는 요소가 되는데 이를 노드라고 한다.

노드의 객체 타입

  • 문서 노드
    DOM 트리 최상위에 존재하는 루트 노드. document 객체를 말한다.
    전역 객체 window의 document 프로퍼티에 바인딩 되어 있다.

  • 요소 노드
    HTML 요소를 가르키는 객체이다. HTML 요소 간의 중첩 관계를 가지고 부자 관계를 가지며 정보를 구조화한다.

  • 어트리뷰트 노드
    HTML 요소의 어트리뷰트를 가르키는 객체이다. 지정된 HTML 요소 노드와 형제 관계를 갖는다.

  • 텍스트 노드
    HTML 요소의 텍스트를 가르키는 객체이다. 요소 노드의 자식 노드이며 DOM 트리의 최종단인 리프 노드이다.

노드의 상속 구조

DOM을 구성하는 노드 객체는 프로토타입에 의한 상속 관계를 갖는다.
Object - EventTarget - Node 가 기본으로 갖는 프로토타입 체인이며 그 자식 관계로 노드에 따라
문서 노드는 Document - HTMLDocument
요소 노드는 Element - HTMLElement - HTML(태그 이름)Element
어트리뷰트 노드는 Attr
텍스트 노드는 CharacterData - Text/Conment
라는 노드 객체의 상속 구조를 갖는다.
이 상속관계에 따라 해당 노드 객체는 브라우저에서 필요한 기능을 제공하는 DOM API 를 사용할 수 있다.

요소 노드 취득

HTML의 구조나 내용, 스타일을 조작하기 위해서는 해당 요소의 노드를 취득해야한다.

  • id 요소로 취득
    Document.prototype.getElementById 메소드를 사용하여 인수로 전달한 id 값을 갖는 하나의 요소를 탐색하여 반환한다.

    • 당연히 id는 중복되지 않는 값이기 때문에 1개만 있어야하지만 만약 동일한 id 값을 갖는 객체가 있다고 해도 에러 없이 해당 요소들 중 가장 첫번째 요소 객체를 반환한다.
    • 해당 id의 요소가 없을 경우, null값을 반환한다.

  • 태그 이름으로 취득
    Document.prototype.getElemenstByTagName/Element.prototype.getElemenstByTagName 메소드를 사용하여 인수로 전달한 태그 이름을 갖는 모든 요소를 탐색하여 반환된다.

    • 탐색된 값은 유사배열 객체인 HTMLCollection 객체로 반환된다.

  • class 요소로 취득
    Document.prototype.getElemenstByClassName/Element.prototype.getElemenstByClassName 메소드를 사용하여 인수로 전달한 클래스 이름을 갖는 모든 요소를 탐색하여 반환한다.

    • 탐색된 값은 유사배열 객체인 HTMLCollection 객체로 반환된다.

  • css 선택자로 취득
    Document.prototype.querySelecter/Element.prototype.querySelecter 메소드를 사용하여 인수에 전달한 css 선택자를 만족시키는 하나의 요소를 탐색하여 반환한다. 없으면 null 값을 반환
    Document.prototype.querySelecterAll/Element.prototype.querySelecterAll 메소드를 사용하면 만족하는 모든 요소를 탐색하여 반환한다.

    css 선택자에는

      • : 전체 선택자
    • (태그 이름) : 태그 선택자
    • #(id 이름) : id 선택자
    • .(클래스 이름) : 클래스 선택자
    • (태그 이름)[속성 이름 = 값] : 어트리뷰트 선택자
    • (부모 태그 이름) (자식 태그 이름) : 후손 선택자 : 부모 밑으로 상속되는 모든 자식
    • (부모 태그 이름) > (자식 태그 이름) : 자식 선택자 : 부모 바로 밑에 있는 자식
    • (형제 태그 이름) + (찾을 형제 태그 이름) : 인접 형제 선택자 : 검색 기준인 형제 요소에서 바로 뒤에 위치하는 인접한 형제
    • (형제 태그 이름) ~ (찾을 형제 태그 이름) : 일반 형제 선택자 : 검색 기준인 형제 요소 뒤에 오는 형제 태그들 모두
    • (태그 이름):(가상 클래스 이름) : 가상 클래스 선택자 : 해당 상태인 태그

    *id로 취득하는 경우를 제외하고

HTMLCollection 객체와 NodeList 객체

DOM API가 검색한 여러개의 노드를 결과값으로 반환하기 위해 만드는 유사배열 객체이다.

  • HTMLCollection 객체
    태그 이름으로 취득하는 경우나 클래스로 취득하는 경우에 반환되는 값으로서 노드 객체의 변화를 실시간으로 반영한다.(살아있는 객체)
    살아있는 객체 문제를 해결하기 위해서 배열로 변환하여 사용하는 것을 추천한다.

  • NodeList 객체
    css 선택자로 취득하는 경우에 반환되는 값이나 자식 요소를 찾는 childNodes 프로퍼티가 반환한 값이다.
    css 선택자로 취득하는 경우에는 노드 객체의 변화를 반영하지 않지만, childNode 프로퍼티로 반환되면 노드 객체의 변화를 반영하는 살아있는 객체가 된다.

노드 탐색

어떤 요소를 취득하면 그 요소를 통해 부모, 형제 자식 요소에 접근할 수 있다. 이렇게
해당 노드는 노드 탐색을 위한 접근자 프로퍼티를 갖는다. 읽기 전용 값이다. 즉, 값을 할당하면 에러 없이 무시된다.

  • parentNode : 부모 요소
  • previousSibling/previousElementSibling : 바로 앞에 있는 형제 요소 (텍스트 노드 포함/미포함)
  • nextSobling/nextElementSibling : 바로 뒤에 있는 형제 요소 (텍스트 노드 포함/미포함)
  • firstChild/firstElementChild : 첫번째 자식 요소 (텍스트 노드 포함/미포함)
  • lastChild/lastElementChild : 마지막 자식 요소 (텍스트 노드 포함/미포함)
  • childNodes/children : 자식 요소들 (텍스트 노드 포함/미포함)

공백 텍스트 노드

HTML 문서에서 태그 사이에 개행문자가 포함되있으면 그 공백을 의미하는 텍스트 노드를 생성한다. 따라서 검색시에 유의해야한다.

자식 노드 확인

Node.prototype.hasChildNodes 메소드로 확인할 수 있다. 하지만 이 메소드는 텍스트 노드를 포함하여 자식 노드의 존재를 확인한다.
따라서 요소 노드만을 확인하고 싶은 경우에는 Node.prototype.children.length 또는 Node.prototype.childElementCount 프로퍼티로 확인하면 된다.

텍스트 노드 확인

해당 노드에 자식으로 텍스트 노드가 연결되어 있기 때문에 firstChild나 childNodes로 접근할 수 있다.

탐색 가능 여부 확인

Element.prototype.matches 메소드로 검색할 특정 노드를 선택자의 형태로 전달하면 확인 가능하다.

노드 정보 취득

  • Node.prototype.nodeType
    노드 객체의 종류를 나타내는 상수를 반환한다.

    • 요소 노드 : 1
    • 텍스트 노드 : 3
    • 문서 노드 : 9

  • Node.prototype.nodeName
    노드 이름을 문자열로 반환한다.

    • 요소 노드 : 대문자 문자열로 태그 이름 반환
    • 텍스트 노드 : ‘#text’ 문자열 값을 반환
    • 문서 노드 : ‘#document’ 문자열 값을 반환

요소 노드 텍스트 조작

  • nodeValue
    텍스트 노드의 값을 반환한다. 이때 호출하는 대상이 텍스트 노드가 아닌 경우 null을 반환한다.

  • textCotent
    해당 요소의 모든 컨텐츠 영역 내에 텍스트를 전부 반환한다. 하위 요소에 텍스트 노드의 값을 전부 반환한다.

DOM 조작

  • innerHTML
    Elemnet.prototype.innerHTML 프로퍼티는 요소 노드의 HTML 마크업을 취득하거나 변경한다.
    단점 : 보안 취약, 추가할 경우 요소 노드를 새로 만듬, 위치를 지정할 수 없음.

  • insertAdjacentHTML
    Elemnet.prototype.insertAdjacentHTML 프로퍼티는 기존 요소를 제거하지 않으면서 위치를 지정해 새로운 요소를 추가한다.
    Elemnet.prototype.insertAdjacentHTML(position, DOM_String)의 형태로 사용한다.

    • position
      beforebegin, afterbegin, beforeend, afterend

    단점 : 보안 취약, 불필요한 요소 노드를 다시 만듬

  • createElement
    Node.prototype.createElement 메소드는 요소 노드를 생성한다. 인자 값으로 준 문자열로 노드를 만든다.

  • createTextNode
    Node.prototype.createTextNode 메소드는 텍스트 노드를 생성한다. 인자 값으로 준 문자열을 값으로 갖는 텍스트 노드를 만든다.

  • createDocumentFragment
    Document.prototype.createDocumentFragment 메소드는 많은 형제 요소를 제어하기 위한 빈 노드를 생성한다. Fragment 객체는 append할 때 사라지기 때문에 좋다.

  • appendChild
    Node.prototype.appendChild 메소드는 인수로 전달한 노드를 해당 노드의 자식 노드로 추가한다. 전달하는 노드를 추가하고 원래 위치의 노드를 제거하면 이동하는 효과를 볼 수 있다.

  • insertBefore
    Node.prototype.insertBefore 메소드는 첫번째 인수로 삽입할 노드를 두번째 인수로 받은 노드 앞에 삽입한다. appendChild와 마찬가지로 삽입할 노드 원본을 지우면 이동하는 효과를 볼 수 있다.

  • cloneNode
    Node.prototype.cloneNode 메소드는 해당 노드의 사본을 만든다. 인수로 블리언 값을 받을 수 있는데 ‘true’일 경우 깊은 복사를, ‘false’일 경우에 얕은 복사를 한다.

  • replaceChild
    Node.prototype.replaceChild 메소드는 호출한 노드의 자식을 다른 노드로 교체한다. 첫번째 인수로 교체할 새로운 노드를 두번째 인수로 교체 될 노드를 전달하면 된다.

  • removeChild
    Node.prototype.removeChild 메소드는 호출한 노드의 자식 중 인수로 전달한 노드를 DOM에서 삭제한다.

어트리뷰트

HTML 요소는 여러개의 어트리뷰트를 가질 수 있다. 이 어트리뷰트도 노드로 변환되어 추가가 된다. 그리고 이 어트리뷰트들은 유사 배열 객체인 NamedNodeMap 객체로 묶여지고 해당 요소 노드의 arrtibutes 프로퍼티에 저장된다. arrtibute 프로퍼티도 읽기 전용임으로 할당시 무시된다.

어트리뷰트의 의미

어트리뷰트 노드는 초기 상태를 관리한다. DOM 노드는 최신 상태를 관리한다.

어트리뷰트 조작

  • getArrtibute
    Element.prototype.getArrtibute 메소드는 해당 요소의 어트리뷰트 값을 취득 및 변경할 수 있다.
    첫번째 인수로 해당 요소의 어트리뷰트를 두번째 인수로 변경할 어트리뷰트 값을 전달할 수 있으며 첫번째 인수만 전달할 시 해당 어트리뷰트 값을 반환한다.

  • hasAttribute
    Element.prototype.hasAttribute 메소드는 인수로 전달한 어트리뷰트가 호출한 요소 안에 있는지 확인하여 블리언 값을 반환한다.

  • removeArrtibe
    Element.prototype.removeArrtibe 메소드는 호출한 요소 안에 인수로 전달한 어트리뷰트를 삭제한다.

스타일

  • style
    HTMLElement.prototpye.style 프로퍼티는 접근자 프로퍼티로서 요소 노드의 인라인 스타일을 취득하거나 변경한다. 참조시 CSSStyleDeclaration 객체를 반환한다.
    CSSStyleDeclaration 객체는 카멜 케이스로 프로퍼티를 갖는다. 케밥 케이스를 그대로 사용하려면 대괄호 표기법으로 참조해야한다.

  • className
    Element.prototype.className 프로퍼티는 접근자 프로퍼티로서 호출한 요소의 class 값을 취득 및 변경한다.

  • classList
    Element.prototype.classList 프로퍼티는 어트리뷰트 값을 모두 담은 DOMtokenList 객체를 반환한다.

    • Element.prototype.classList.add
      인수로 전달한 class를 추가한다.
    • Element.prototype.classList.remove
      인수로 전달한 class를 제거한다.
    • Element.prototype.classList.index
      인수로 전달한 인덱스에 해당하는 class 이름을 문자열로 반환한다.
    • Element.prototype.classList.contains
      인수로 전달한 문자열에 해당하는 class 이름이 있는지 확인한다.
    • Element.prototype.classList.replace
      첫번째로 전달한 문자열의 class 이름을 두번째 문자열의 class 이름으로 바꾼다.

  • getComputedStyle
    인라인 스타일만 반환하는 style 프로퍼티와 다르게 window.getComputedStyle 메소드는 인수로 전달한 요소의 적용된 모든 스타일을 CSSStyleDeclaration 객체에 담아 반환한다.
    모든 스타일이란 링크 스타일, 임베딩 스타일, 인라인 스타일, 외부에서 적용된 스타일, 상속된 스타일, 사용자 지정 스타일 등을 말한다.
